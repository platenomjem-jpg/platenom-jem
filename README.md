# نظام إدارة النادي الرياضي والكافيه

نظام شامل لإدارة نادي رياضي مع كافيه: عضويات، حضور، نقطة بيع، مخزون، وحسابات.

نفس تقنيات مشروع **جود العراق**: Node.js + Express + PostgreSQL + HTML/CSS/JS عادي، جاهز للنشر على Railway.

## الموديولات

| الموديول | الوصف |
|---|---|
| 👥 الأعضاء | تسجيل أعضاء، خطط اشتراك، تجديد، تتبع الحضور |
| ☕ نقطة البيع (POS) | بيع منتجات الكافيه، خصم تلقائي من المخزون |
| 📦 المخزون | إدارة المنتجات، حركات وارد/صادر/تسوية، تنبيه نفاد |
| 💰 التقارير | إيرادات (كافيه + اشتراكات)، مصروفات، صافي الربح |
| 🔐 المستخدمين | صلاحيات (مدير / استقبال / كاشير / محاسب) |

## التشغيل محلياً

```bash
# 1. تثبيت الاعتماديات
npm install

# 2. إنشاء ملف .env (انسخ من .env.example) وعدّل DATABASE_URL
cp .env.example .env

# 3. تهيئة قاعدة البيانات (إنشاء الجداول + مستخدم أدمن افتراضي)
npm run init-db

# 4. تشغيل السيرفر
npm start
# أو في وضع التطوير مع إعادة التشغيل التلقائي:
npm run dev
```

السيرفر يعمل على `http://localhost:3000`

**بيانات الدخول الافتراضية بعد init-db:**
- اسم المستخدم: `admin`
- كلمة المرور: `admin123`
- ⚠️ **غيّرها فوراً بعد أول تسجيل دخول**

## النشر على Railway

1. أنشئ مشروع جديد على Railway واربطه بمستودع GitHub
2. أضف خدمة PostgreSQL من Railway (يولّد `DATABASE_URL` تلقائياً)
3. أضف متغيرات البيئة: `SESSION_SECRET`, `JWT_SECRET`, `NODE_ENV=production`
4. شغّل أمر `npm run init-db` مرة واحدة من Railway CLI أو من تبويب Shell
5. Railway يشغّل `npm start` تلقائياً

## هيكل المشروع (بدون مجلدات فرعية - سهل الرفع على GitHub)

```
sports-club-cafe/
├── server.js              # نقطة الدخول الرئيسية
├── db.js                  # الاتصال بـ PostgreSQL
├── schema.sql             # تعريف كل الجداول
├── initDb.js              # سكربت تهيئة قاعدة البيانات
├── auth-middleware.js     # حماية المسارات + الصلاحيات
├── route-auth.js          # تسجيل دخول/خروج
├── route-members.js       # الأعضاء + الاشتراكات + الحضور
├── route-products.js      # المنتجات + المخزون
├── route-orders.js        # نقطة البيع (POS)
├── route-reports.js       # التقارير + المصروفات
├── package.json
├── .env.example
└── public/                 # مجلد واحد فقط (مطلوب من Express لتقديم الواجهة)
    ├── index.html
    ├── style.css
    ├── api.js, app.js, auth.js, dashboard.js, members.js, products.js, pos.js, reports.js
```

ملاحظة: مجلد `public/` ضروري وموجود لأن Express يحتاج مجلداً واحداً محدداً لتقديم ملفات الواجهة (HTML/CSS/JS) للمتصفح — هذا غير المجلدات الفرعية المتعددة (config/routes/middleware) اللي كانت موجودة بالنسخة الأصلية والآن أصبحت كل ملفاتها بجذر المشروع مباشرة.

## ملاحظات تقنية مهمة

- جميع عمليات المخزون والبيع تستخدم **transactions** (BEGIN/COMMIT/ROLLBACK) لمنع تضارب البيانات.
- الجلسات (sessions) تُحفظ في PostgreSQL نفسه (`connect-pg-simple`) فتبقى فعالة حتى بعد إعادة تشغيل السيرفر.
- لإضافة أعمدة جديدة بأمان مستقبلاً: استخدم `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` بدل تعديل `schema.sql` مباشرة على بيانات حية (نفس النهج المستخدم بمشروع جود العراق).
- الصلاحيات حالياً 4 أدوار: `admin`, `receptionist`, `cashier`, `accountant` — يمكن توسيعها بسهولة من `middleware/auth.js`.
